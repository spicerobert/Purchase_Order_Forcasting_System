# 採購預測系統 Purchase Order Forecasting System

依過往需求量、即時庫存與需求分類，自動計算安全庫存、補貨點與建議採購量。

---

## 專案效益

- **取代人工試算**：自動從 ERP 擷取 52 週歷史需求與即時庫存，不再手動整理報表
- **需求智慧分群**：將每個品號依統計特徵分為四種模式，對症下藥計算採購量
- **降低缺貨與過剩**：以安全庫存公式與補貨點，在正確時機提出正確數量的採購建議
- **結果即時回寫**：計算完成後直接更新 Google Sheets，供採購人員直接參考

---

## 系統流程

```
Google Sheets 品號參數
  （品號、服務水準、前置週數、最小採購量…）
        │
        ▼
  查詢 ERP（MS-SQL）
  ├─ 52 週週需求歷史（INVLA）
  ├─ 現有庫存（INVLC + INVLA）
  └─ 在途庫存（PURTD 已確認未到貨）
        │
        ▼
  需求分群（A / B / C / D）
  ├─ A：全年穩定（CV ≤ 0.5，淡旺季比值接近 1）
  ├─ B：日曆季節性（旺季/淡季比 ≥ 1.5，波峰落在中秋/春節前後）
  ├─ C：不規則波動（有波峰但不符合日曆錨點）
  └─ D：零星低頻（零需求週 ≥ 35% 或週平均 ≤ 3 件）
        │
        ▼
  決定採用週需求 D
  ├─ B 模式：依目前是否處於旺季，取旺季或淡季平均
  ├─ A / C 模式：取 52 週年均值
  └─ D 模式：年均值 × 趨勢係數 adj
        │
        ▼
  趨勢調整（adj = 近 4 週均 / 再往前 4 週均，限縮於 0.5～1.5）
        │
        ▼
  安全庫存計算
  SS  = Z × √(STD² × L + STD_L² × D²)
  ROP = D × L + SS
  建議採購量 = max(0, (L + T) × D + SS − 現有合計庫存)
              ↑ 無條件進位至最小採購量倍數
        │
        ▼
  產生進貨建議文字
  庫存0 / 需要進貨 / 庫存偏高 / （正常則空白）
        │
        ▼
Google Sheets 進貨建議（22 欄輸出）
```

> 品號設為「有單才進」（MTO）者，略過 SS/ROP 計算，建議欄顯示「依訂單」。

## 快速開始

```bash
# 1. 安裝相依套件（需安裝 uv）
uv sync

# 2. 複製設定檔並填入 ERP 連線資訊與 Google Sheets URL
cp config.ini.template config.ini

# 3. 放置 Google 服務帳號憑證
#    將 service_account.json 放至專案根目錄

# 4. 初始化 Google Sheets 工作表結構（僅需執行一次）
uv run setup_sheets.py

# 5. 在「品號參數」工作表填入品號後，執行主流程
uv run main.py
```

其他指令：

```bash
uv run run_demand_profile.py   # 需求分群分析，輸出至 Sheets 與 CSV
uv run query_inventory.py      # 快速查看現有 + 在途庫存（不執行計算）
```
